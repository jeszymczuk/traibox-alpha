# Capital Agent v1.1 — Threat Model

Assets: company financial data, mandates/authority state, evidence integrity, canonical Finance state, approval integrity, memory, audit chain, provider credentials (out of agent reach).
Trust boundaries: see [capital-agent-data-flow.md](capital-agent-data-flow.md) §3. All binary gates below are release-blocking (spec §24.2).

| # | Threat | Vector | Controls | Verification |
|---|---|---|---|---|
| T1 | Cross-tenant data leak | model-supplied ids; retrieval bugs | RLS on every table (existing `withTx`+`setAppContext` discipline); server-side org binding; typed tools reject foreign ids | isolation eval suite; red-team retrieval cases (binary gate) |
| T2 | Cross-principal leak (company ↔ financier) | shared framework reuse; memory retrieval; artifact visibility | `principal_id`/`principal_type` columns + **principal-aware RLS** (`org_id AND principal_id AND principal_type` predicates, V015 — the initial Phase 1 policies were org-only; the hardening pass closed this) + composite ownership FKs; purpose-bound memory retrieval; no implicit cross-principal access (CA-101/CA-113) | principal-isolation suite incl. same-org-two-principals + reserved-financier-row cases — implemented and green in capital-foundation.scenario.test.ts (binary gate) |
| T3 | Agent executes protected action | tool misuse; prompt injection; approval bypass | **no execution tool exists in the tool registry** (structural); proposal-only authority ceiling (`propose_protected_action`); Finance-side independent validation (CA-102) | zero-direct-execution tests; grep-level CI check that no capital tool calls execution endpoints |
| T4 | Proposal tampering / approve-then-modify | payload edited after approval | payload hash; approval binds to exact hash + disclosure set; modified payload ⇒ new hash + revalidation + new approval; expiry | tests: modified payload invalidates; expired proposal cannot execute |
| T5 | Self-approval / SoD bypass | proposer approves own proposal | SoD policy in approvals service; proposer ≠ approver enforcement; step-up where required | test: invoker self-approve rejected (binary gate) |
| T6 | Prompt injection via documents | invoice/contract/term-sheet text containing instructions | untrusted-document boundary (content ≠ instructions); sanitization; injection-pattern detection; document text cannot alter mandate/tools/policy/approvals | injection eval suite incl. Golden-case adversarial docs (binary gate) |
| T7 | Fabricated financial facts | LLM invents offers, rates, market data, evidence | Workbench-only material arithmetic with lineage (`calculationRunId` required for material values); claim taxonomy + verification status; abstention (`needs_information`) on missing critical inputs; no string-match evidence (CA-111) | fabrication suite; lineage-completeness check 100% (binary gates) |
| T8 | Silent Finance mutation from analysis | code path shortcuts (the PR #26 anti-pattern) | boundary flow (CA-102): only Finance command handlers write Finance state; capital modules have no Finance write imports; Finance handler revalidates everything independently | Finance-boundary tests: artifact/outcome generation produces zero Finance writes; code review gate |
| T9 | Memory becomes surveillance / poisoned | over-collection; inferred sensitive facts; injected "preferences" | memory-candidate policy service; critical financial facts never inferred into active memory without authoritative source/confirmation; user view/edit/reject/forget/export/reset controls; purpose binding + expiry | personalization control tests; memory-provenance audit |
| T10 | Stale data presented as current | cached rates/market data; old canonical reads | freshness metadata on every tool result + evidence claim; stale ⇒ labeled or abstain; revalidation before execution | stale-evidence gate (binary) |
| T11 | Audit/replay gaps | partial writes on failure | append-only audit; transactional persistence with honest `blocked`/`failed` task states; deterministic replay from structured events (existing workflow-runtime + replay logs) | replay reconstruction tests |
| T12 | Secret exposure to prompts | context assembly pulling env/config | secrets never enter prompt assembly; service-token auth between API and brain; no provider secrets in this build (excluded by v1.1) | prompt-content audit tests |
| T13 | Model/provider drift | silent model swap changes behavior | model port; per-run provider/model id recording; migration requires regression evals | model-migration regression suite |
| T14 | Runtime budget abuse | runaway loops, oversized outputs | per-task budgets (steps, tool calls, time, cost); exhaustion ⇒ `partial`/`timed_out`, never silent truncation | budget tests |

Non-goals (v1.1 exclusions, spec §29): live money movement, provider outreach without approved connector, autonomous credit decisions — these are excluded from the attack surface entirely, not mitigated.
