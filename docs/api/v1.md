# TRAIBOX API v1

TRAIBOX exposes its alpha API contract from the same shared package used by the web app, API, tests, and future SDKs.

## Discovery

- `GET /v1/api/catalog` returns the versioned endpoint catalog, error taxonomy, idempotency policy, trace header, and org header.
- `GET /v1/openapi.json` returns an OpenAPI 3.1 document generated from the contract catalog.
- The source of truth is `TRAIBOX_API_ENDPOINTS` in `@traibox/contracts`.

## Required Headers

- `Authorization: Bearer <token>` is required for user-scoped routes.
- `X-Org-Id: <uuid>` is required for org-scoped routes.
- `X-Trace-Id: <string>` is optional. If omitted, the API generates one.
- `X-Idempotency-Key: <string>` is required for externally consequential write routes listed in the API catalog.

## Error Envelope

All API errors should return the shared envelope:

```json
{
  "error": "missing_idempotency",
  "message": "X-Idempotency-Key is required",
  "hint": "Send X-Idempotency-Key for protected or externally consequential write routes.",
  "trace_id": "trc_example",
  "status_code": 400,
  "category": "idempotency",
  "retryable": false,
  "docs_url": "/v1/api/catalog#errors-missing_idempotency"
}
```

## Release Gate

Before promoting an alpha build, run:

```sh
pnpm release:gate
```

CI additionally runs real Postgres alpha scenario tests and persists Trade Brain eval artifacts.
